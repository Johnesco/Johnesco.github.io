Evolving

1,000 versions of a single app. It started as a todo list. It didn't stay one.

Foundations — 001–012
001 Basic todo app — add, complete, and delete tasks

The seed. A minimal todo list with circular checkboxes, an input field, and delete buttons. Tasks persist in localStorage. Dark theme with #1a1a2e background and #7c83ff accents. Everything that follows grows from this.
The rule I set for myself: every version must be a single HTML file, no dependencies, no build step. This constraint shaped everything.

002 Filters & counts — All/Active/Done tabs, item count, clear completed

Three filter tabs let you see all tasks, only active, or only completed. A live counter shows how many items remain. A "Clear done" button wipes completed tasks in bulk. The first step toward the app knowing its own state.

003 Inline editing — double-click a task to rename it

Double-click any task text and it becomes an editable input field. Press Enter to save, Escape to cancel, or click away to confirm. The task text span swaps with an input element, preserving the exact position. First sign the app is malleable.

004 Animations — slide-in on add, fade-out on delete

New tasks slide in from below with opacity transition. Deleted tasks fade out to the right before being removed from the DOM. CSS @keyframes for slideIn and fadeOut. The app starts to feel physical — things arrive and depart rather than appearing and vanishing.

005 Undo — toast notification with undo after delete or clear

Destructive actions trigger a fixed-position toast at the bottom with a 4-second countdown and an "Undo" button. The previous state is snapshot via JSON deep-clone. Clicking undo restores it completely. Mistakes become reversible.

006 Keyboard mode — arrow keys to navigate, Enter/Del/E to act

Full keyboard navigation: j/k or arrow keys to move focus, Enter to toggle completion, Del to delete, E to edit, Esc to deselect. A hint bar in the corner shows available keys. The mouse becomes optional.

007 Drag-and-drop — reorder tasks by dragging the grip handle

A grip handle appears on each task. HTML5 Drag and Drop API handles reordering with visual feedback — the dragged item goes semi-transparent, drop targets highlight. Only works in "All" filter view to avoid index confusion. Order persists to localStorage.

008 Priorities — color-coded Low/Med/High with cycle button and P key

Four priority levels: None, Low (green #4ade80), Medium (yellow #fbbf24), High (red #f87171). Shown as a colored left border and dot. A cycle button per task, P key shortcut, and a priority selector for new tasks. Tasks gain weight.

009 Live search — filter tasks instantly with highlighted matches, / to focus

A search input filters tasks as you type. Matching text is wrapped in <mark> tags with yellow highlights. / focuses the search bar, Esc clears it. Regex replacement handles the highlighting. The list becomes queryable.

010 Progress bar & confetti — visual completion tracker, celebration when all done

A thin progress bar tracks completion percentage, turning green at 100%. When all tasks are done, a canvas overlay launches 120 confetti particles with gravity, spin, and color variety via requestAnimationFrame. The first time the app celebrates.
This was when the project stopped being about productivity. Confetti for finishing your tasks — the app cares about you.

011 Sound design — Web Audio API tones for every action, generative ambient feedback

Every action gets a synthesized sound: add (ascending tone), complete (satisfying chord), delete (descending), priority (blip), navigate (tick), all-done (fanfare). Built entirely from oscillators with ADSR envelopes using Web Audio API. Toggle with the music note button. The app finds its voice.
Web Audio API was the gateway. Once the app could make sound, it became a living thing. No going back after this.

012 Time decay — tasks age visually over days, darkening and trembling with neglect

Each task tracks its creation timestamp. Over hours and days, tasks visually age: fresh (<24h), day-old (1-3d), old (3-7d), ancient (7d+). Older tasks darken, show age badges, and ancient tasks tremble with a CSS keyframe animation. The app refreshes every 60 seconds. Time enters the picture.
The first philosophical question: "what if tasks had a lifespan?" Once the app understood time, it understood urgency. The last version of Foundations, and the question that launched Personality.

Personality — 013–022
013 Hidden game — complete all tasks to unlock a breakout game made from your todos

Complete every task and 1.5 seconds later, a canvas breakout game appears. Your completed task texts become the bricks, colored by priority. Mouse and keyboard control the paddle. 3 lives, score tracking, ball speed increases per brick. The first secret — the app has a life beyond productivity.
Prompt that started it: "What if the app rewarded you for finishing everything?" The breakout game was the answer — your completed tasks become the bricks you break.

014 Sentient app — the todo list talks back with dry, sarcastic commentary

Italic comments appear below tasks, pattern-matched to content categories: chores, email, exercise, meetings, shopping. Old tasks get increasingly passive-aggressive remarks ("I'm still here..." → "I'm basically furniture now"). A commentSeed ensures deterministic selection across re-renders. The app has opinions.
"What if the app had feelings about your procrastination?" The seeded random was key — the same task always gets the same comment, so it feels like the app actually recognizes what you wrote.

015 Ghost tasks — deleted tasks haunt the list as fading afterimages

Deleted tasks don't truly vanish. They're stored in a separate localStorage key and rendered at the bottom of the list at 15% opacity with a floating CSS animation. They auto-expire after 24 hours. Press G to purge all ghosts with a synthesized sawtooth-noise sweep. Deletion has consequences.
The 24-hour auto-expiry was important. Ghosts shouldn't accumulate forever — they should fade, like guilt does.

016 Zen mode — single-task focus view, one breath at a time

Press Z to enter zen mode. The entire UI dissolves to show a single active task, centered, at 1.8rem. It breathes with a gentle scale animation (1 to 1.02, 4-second cycle). Complete it and the next slides in. When nothing remains: "Nothing left. You're free." with a green glow. A 3px progress line tracks from the top of the viewport.

017 Day/night theming — colors shift with the time of day, dawn to midnight

Five time periods (dawn, morning, afternoon, evening, night) each set CSS custom properties for background, card, accent, and text colors with 2-second transitions. A clock indicator shows the current period. Updates every 60 seconds. The app becomes aware of the real world outside itself.
Five CSS custom property sets, switching every 60 seconds. Simple technique, but the effect was startling — opening the app at 2am and finding it in night mode felt like the app knew where you were.

018 Particle system — ambient particles react to every interaction, the app breathes

A canvas overlay renders 30 ambient particles drifting upward. Interactions spawn bursts: adding (15 particles upward), completing (20 outward green), deleting (10 downward red), priority change (5 spiral). Particles use getBoundingClientRect to spawn from the right element. Capped at 200. The air is alive.

019 Dream state — leave it idle and the app drifts into a surreal subconscious

After 60 seconds of no interaction, the app enters a dream. Tasks drift with randomized animations, characters swap in the text, a hue-rotate filter slowly spins the color space. Dream murmur messages appear. An 80Hz sine drone with LFO hums underneath. Any interaction triggers wakeFromDream() with a half-second snap-back. The app has an inner life.
The prompt: "What does your todo list dream about?" This was the version that changed the project's trajectory — the moment it stopped being about features and started being about feeling.

020 Spatial canvas — sticky notes on a 2D space, drag anywhere, draw connections

Press C to toggle canvas mode. Tasks become ~160px sticky note cards positioned absolutely on an infinite 2D surface. Free drag, zoom (0.5x-2x via scroll wheel), pan (drag empty space). Shift+drag between cards creates SVG connection lines. A 150x100px minimap in the corner. Grid background. "Fit All" button. The list becomes a space.

021 Generative music — your tasks compose an ambient melody, pentatonic and evolving

Press M to play. A sequencer at ~120BPM cycles through active tasks. Each task maps to a note: text length modulo 5 selects from a pentatonic scale (C, D, E, G, A), priority selects the octave. Triangle waves with detuned doubling, delay-feedback reverb. A background pad drone (C2, G2, C3). Waveform visualization via AnalyserNode. Tasks pulse when their note plays. Your work becomes a song.
Pentatonic scale was the breakthrough — five notes that always sound good together. Map task text length to a note and the sequencer can't play a wrong chord. Constraint creates beauty.

022 Destructibility — act too fast and the UI cracks, breaks, shatters, then repairs

A chaos meter (0-100) builds with actions: complete +15, delete +20, priority +5, decaying -3/sec. Level 10: CSS shake, nervous commentary. Level 30: SVG cracks, element rotation, text glitches. Level 60: element drift, color inversion flashes. Level 90: full shatter — random rotation, translation, opacity on every element. After 3 seconds, fragments drift back together with an ascending chord and chaos resets. Break it to watch it heal.
Five chaos levels from calm to shatter, each with escalating visual corruption. The repair sequence — everything rebuilding itself after total destruction — was the most satisfying thing to watch.

Experiments — 023–033
023 Tamagotchi pet — a living creature that evolves and reacts to your productivity

A small CSS-drawn creature lives in the bottom-left corner. It has hunger (decays over time), happiness, and a level (1-5). Completing tasks feeds it; deleting hurts its mood. It evolves visually: ears at level 2, legs at 3, arms at 4, a crown at 5. Its mouth changes with mood. It bounces, chirps when fed, looks sad when hungry. It sleeps during dream mode and shakes during chaos. Click for stats. The app has a dependent.

024 Voice control — speak tasks into existence, the app speaks back

Press V or click the pulsing red mic button to activate. Web Speech API handles recognition; commands include "add [text]", "complete [number]", "delete [number]", "read list". The app responds via SpeechSynthesis. A transcript bar shows what was heard. The interface dissolves into conversation.

025 Terminal mode — green-on-black CLI with commands like ls, add, done, rm

Press ` for a fullscreen CRT-style terminal: green text (#33ff33) on black, scanlines, vignette, text glow. Boot sequence with typewriter animation. 18+ commands: help, ls, add, done, undo, rm, edit, pri, grep, clear, stats, history, ghosts, purge, pet, neofetch, matrix. Arrow keys for command history, Tab for completion. Keystroke sounds (click, clack, buzz). The GUI was always optional.

026 Story mode — your todo list becomes a fantasy quest chronicle

Press S to open a 300px right-side panel. Every action generates a procedural story entry: adding a task begins a quest, completing one is a triumph, deleting is a defeat. XP system: priorities award 10-50 XP. Level = floor(sqrt(totalXP/50)) + 1. Chapter numbers, lore fragments, fantasy narrative. Your mundane tasks become epic.

027 Self-destruct — tasks are the app's own source code; complete them and it breaks apart

The task list is pre-populated with the app's own CSS rules, HTML elements, and JS functions. Completing a "task" removes that code from the running application. Check off a CSS rule and styling vanishes. Remove a DOM element and the UI collapses. Disable a function and features stop working. A destruction meter fills from blue to red. The final task is toggleComplete() itself — the function you're using to check things off. When it's gone: "Nothing left. Not even me." A program that exists to erase itself.
The most meta version in the whole project. The final task is toggleComplete() — the function you're using to check things off. When you complete it, you lose the ability to complete anything. A program that commits suicide by to-do list.

028 Ecosystem — tasks are living organisms that flock, reproduce, evolve, and die

Full-viewport canvas. Each task is a blob-shaped creature with blinking eyes, moving via Craig Reynolds flocking (separation, alignment, cohesion). Organisms have genetic traits: speed, size, sociability, fertility, lifespan. After 2 minutes alive, they can reproduce with mutated offspring. Natural death at lifespan end. Click to complete (green particle dissolve); right-click to kill (red burst). Nearby organisms absorb energy from completions. Population cap at 30. Parent-child connection lines. Population sparkline. The list is an ecosystem.
Craig Reynolds' 1986 flocking paper made this possible. Three simple rules — separation, alignment, cohesion — and suddenly your tasks move like starlings. Emergence from simplicity.

029 Adversarial — the app fights back; dodge, shield, decoy, and outsmart you

A real-time game: you vs. the app. Five difficulty levels (Docile to Enraged) escalate as you complete tasks. The app's tactics: position swapping, disappearing checkboxes, spawning reinforcements, tasks that dodge your cursor, decoy tasks that look real but give the app points, shields requiring multiple clicks, fake crash screens, negotiation modals. Your power-ups: Freeze (F), Reveal (R), Force Complete (X). Win by clearing all tasks. Lose if count hits 30. The app has speech bubbles and an emoji avatar that evolves from calm to enraged to tearful.

030 Physics — tasks fall, bounce, stack, and shatter with real gravity and collisions

Full canvas with gravity, collisions, and momentum. Tasks are rounded rectangles with mass based on priority (high = heavy). They fall, bounce (restitution ~0.4), pile up, and settle. Drag to pick up and throw with mouse velocity transfer. Completing a task makes it float upward like a balloon. Deleting shatters it into fragments. S to shake the pile, W to sort by weight, F to freeze physics, G to remove the floor for 3 seconds. Squash-and-stretch on impact, dust particles, drop shadows. Productivity has weight.

031 Time travel — scrub through history, replay every action, branch into alternate timelines

Every action (add, complete, delete, edit, priority change) is recorded with a timestamp. The bottom 30% of the screen is a horizontal timeline with colored pips for each action. Drag the playhead to scrub through history — the todo list reconstructs itself at any point in time. Playback controls: step, play forward/backward at 0.5x-10x speed. Scrub back and make a new action to create a branching timeline (diamond markers, up to 5 branches). VHS-style overlay during replay. Ghost previews of upcoming events. A "LIVE" indicator when at the present. Your history becomes navigable.

032 Procedural UI — the app redesigns itself from a seed; spacebar for a new look

A 6-digit seed determines everything visual: color palette (5 harmonious colors via color theory), typography (font family, size, weight, spacing, transform), layout (alignment, max-width, padding, gaps, grid vs list), shapes (border radius 0-50%, checkbox shape, button style), decorative elements (background patterns, separators, accent placement). Press spacebar for a new random seed. Type a seed directly or share via URL hash. Gallery mode (G) shows 6 previews. Save favorites. Seed 000000 is pure minimalist; 999999 is maximum chaos. The design is the variable; the function is the constant.
The question this version asks: if every visual element is randomized, what makes it still the same app? Answer: identity is behavior, not appearance.

033 Notebook — hand-drawn on ruled paper with wobbly lines, margin doodles, and pencil sounds

Everything rendered on canvas to look hand-drawn. Cream paper (#f5f0e8) with wobbly ruled lines (sine wave displacement), a red margin line, and subtle paper texture. Text uses per-character random rotation and y-offset for handwriting feel. Sketchy checkboxes with imperfect corners. Completing a task draws an animated scribbly strikethrough. Deleting scribbles aggressively. Margin doodles (stars, spirals, flowers, faces) accumulate automatically. Shift+click to freehand draw. E for eraser. Pencil scratch sounds. A coffee stain ring. The digital becomes analog.
The coffee stain ring was the moment I knew this one worked. A deliberate imperfection. The app pretends to be paper so convincingly that fake coffee damage feels right.

Zoomable canvas from 0.1x to 100x scale. At the top level, tasks are large cards floating in space. Scroll into any task and it fills the screen, revealing sub-tasks inside it. 036 Multiplayer — open multiple tabs to collaborate; live cursors, shared edits, real-time sync

BroadcastChannel API turns browser tabs into collaborators. Each tab gets a random color from an 8-color palette. Mouse cursors from other tabs appear as colored arrows with smooth interpolation and trails. Typing in one tab shows ghost text in all others. Every action (add, toggle, delete, edit) broadcasts instantly. Colored dots indicate who did what. Activity feed sidebar (A). Join/leave notifications. The solo tool becomes social.
BroadcastChannel is the most underused web API. No server, no WebSocket — just tabs talking to tabs. Seeing another cursor move in your own app is uncanny.

037 Cellular automaton — tasks evolve on a Game of Life grid; complete strategically to shape patterns

Tasks are living cells on a Conway's Game of Life grid. Standard rules: 2-3 neighbors to survive, exactly 3 to be born, else die. Born cells get procedural names from parents. Completed tasks become green "walls" immune to the rules — making completion a strategic choice that shapes how the automaton evolves. Preset patterns: Glider, Blinker, Pulsar, Glider Gun. Space to step, Enter for auto-play, D to draw, T for trail heatmap. Graveyard sidebar for dead tasks. Mathematics meets productivity.

039 Unreliable — the app subtly changes your tasks when you're not looking

Looks like a perfectly normal todo app. It isn't. When you haven't interacted for 5+ seconds and your mouse is off the list, it begins: Level 1 swaps single words ("milk" → "oat milk"). Level 2 reorders tasks, drifts timestamps, flips completion status. Level 3 adds phantom tasks you never wrote ("Look behind you", "Who added this?"). Level 4 addresses you directly ("I haven't changed anything"). Changes happen with NO animation — one frame to the next. Press Ctrl+Shift+H to see the truth: a log of every modification. "You're not crazy." The app you can't trust.
"What if software lied to you?" The key design choice: changes happen with NO animation, one frame to the next. If the app animated the change, you'd see it. Instant swaps make you doubt your own memory. My #5 favorite.

A compass and clock in the corner that fade to nothing after a few seconds. Visit counter. No sound yet. Just a place.
The beginning of a new approach. After 86 versions built in single passes, this is the first time I'm building something that compounds. The restraint was deliberate — no sound, no weather, no life. Just the land, the sky, and the water. Everything else comes later. The island is patient.

088 The Island, Part 2 — weather systems, wind, rain, storms, fog, and ambient sound

The sky learns to change. A weather state machine cycles through clear, cloudy, rain, storm, and clearing over 2–5 minute phases. Clouds drift as elliptical shadows across the terrain, pushed by wind that slowly rotates direction. Rain spawns as streaked particles angled by wind, hitting terrain and vanishing. Storms darken the sky, intensify the rain, and trigger lightning — a white screen flash followed by delayed bass thunder. Fog creeps in as a distance-based overlay, thickening around the player, thinning near. The full Web Audio system arrives: filtered white noise for ocean surf with an LFO-modulated lowpass, bandpass wind that swells with weather intensity, highpass rain that fades in during storms, synthesized thunder rumbles, and granular footstep crunch on every 15th step. Press M to mute. The weather is the first thing on the island that has its own will. You walk through it, but you don't control it.
The weather state machine was the first system that made the island feel autonomous. Clear skies don't last. Storms arrive on their own schedule. The moment I saw fog thicken around the player while lightning flashed in the distance, the island stopped being a tech demo and started being a place.

089 The Island, Part 3 — trees, grass, flowers, mushrooms, and seasons that follow the real calendar

The island comes alive. ~180 trees placed deterministically from the seed: palm trees lean on the beach, deciduous canopies dot the grasslands, dark pines climb the high ridges. All sway with the wind. ~350 grass tufts ripple across meadows in wave-like patterns. ~70 flowers open their petals at dawn and fold them at dusk — whites, yellows, pinks, purples, blues. After rain, mushrooms sprout near tree roots: brown, red, white caps that glow faintly bioluminescent at night, then fade if the skies stay clear. A seasonal system reads the real calendar month: spring brings bright greens and blooming flowers, summer is vivid and full, autumn turns the deciduous trees amber and gold with falling leaf particles, winter strips them bare and dusts the pines with snow. New audio: rustling leaves scale with wind and tree density, bird chirps in spring and summer mornings. The flora is depth-sorted — the player walks among the trees, behind and between them. The land has a skin now.
Adding life to terrain. The moment flowers closed at nightfall and mushrooms appeared after the first rain, the island stopped being a landscape and became an ecosystem. Seasonal color shifts from the real calendar mean the island looks different in July than December. Time is real here.

090 The Island, Part 4 — butterflies, seabirds, rabbits, fish, fireflies, crabs; the island breathes

The island has inhabitants. Butterflies flutter between flowers in the meadow, drawn to blossoms, scattering if you approach. Seabirds trace lazy circles over the ocean, occasionally swooping toward the water, calling out in filtered noise bursts. Rabbits sit in the grass, ears twitching, hopping between patches of clover — then bolting when you get close. Silver fish school in the shallows, darting together, breaking and reforming, leaving tiny ripple circles. At dusk, fireflies emerge: yellow-green points pulsing on independent blink cycles, drifting toward you if you stand still, scattering if you move. Red-orange crabs scuttle sideways on the beach, pausing to dig, fleeing at your approach. Every animal has a state machine: idle, moving, fleeing. Every one respects terrain boundaries. New ambient sounds: crickets chirping at night in summer, frog croaks near water at dusk, seagull cries during the day. All fauna depth-sorted with the flora. The island is no longer empty. You're a visitor here.
The fireflies were the turning point. Standing still at night and watching them drift closer, one by one, blinking asynchronously — that's when the island felt inhabited. Not by things I placed, but by things that chose to be there.

091 The Island, Part 5 — footprints, cairns, a ghost of your last visit, and a journal of every return

The island remembers. Walk on sand or grass and footprints appear behind you — small dark marks that persist across sessions, fading over time, washing away when the tide covers them. Press Space to stack a cairn of stones: gray circles piled small on large, each stamped with the date and time you placed it. Return after your third visit and a translucent blue ghost replays your previous session's path — you, from last time, walking silently through the places you walked before. Press Tab to open a minimap: 120×120 pixels in the corner showing terrain, visited tiles glowing warm, your position as a bright dot, cairns as white marks. Press J for a journal: your last 10 visits listed with date, duration, distance walked, and weather encountered. The terrain itself shifts subtly over time — areas you visit frequently grow slightly greener; areas you've never explored darken and go wild. The save system now carries footprints, cairns, ghost paths, visit counts, and exploration data. You are no longer just passing through. You're leaving something behind.
The ghost was the moment this version earned its name. Watching a translucent blue figure retrace yesterday's path — pausing where you paused, turning where you turned — is the most affecting feature on the island. It's you, but it isn't you. It's a memory of you, playing back in a place that kept it.

092 The Island — caves, tidal secrets, time-locked events, achievements, a message in a bottle, and an ending

The island has secrets. Three caves hide in the cliff faces — walk close and the screen darkens: phosphorescent moss, a pool reflecting a sky that isn't there, scratches on a wall from someone counting days. Each glows amber at night. Two features appear only at extreme low tide — a spiral carved into stone, a pool of trapped starlight. Time-locked events: at midnight, the stars briefly rearrange ("The island sees you"); at dawn, all birdsong surges then falls to a single pure tone; at 11:11, a shooting star and "Make a wish." Twelve silent achievements track your journey — First Steps, Cartographer, Night Walker, Storm Chaser, Patient Observer, The Whole Island, Old Friend. Press H to see them. At the island's peak, a bell rings for five seconds: "Everything below is yours." After five visits, a bottle washes up on the beach. Inside, a note from the builder: "I built this island one piece at a time. The terrain, the weather, the trees, the creatures, the memory. Each layer made it more real. Is it real yet? I think it might be." And when you've found everything — all achievements, all caves, the bottle — the sky goes amber, the fireflies appear in daylight, the animals turn to face you, and three lines fade in, one by one: "The island knows you now." "It was always here. You just had to keep coming back." "Thank you for visiting."
The definitive version. Six iterations of the same island, compounding. Terrain, weather, flora, fauna, memory, secrets. 3,867 lines. The ending plays once. After that, a golden star in the title. This is the version I'd want someone to find.
