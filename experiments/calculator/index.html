Step 1 7-Segment LCD Display
The calculator will have a display of 20 digits, they will imitate 7 segment LCD screens with internal shadows and transparency to make it more believable.

What was built

  • Created index.html with a dark calculator shell and rounded bezel
  • A green-tinted LCD panel with gradient background and inset shadows to look recessed into the body
  • 20 digit cells, each containing 7 CSS-shaped segments (a–g)
  • Segments use clip-path to create the classic hexagonal LCD segment shape
  • "Off" segments are faintly visible at 12% opacity — just like a real LCD where you can see the inactive segments through the glass
  • "On" segments render at 85% opacity with inset box-shadow for depth
  • Each digit has an accompanying decimal point pip
  • A renderDisplay(str) function parses any string into right-aligned 7-segment output
  • Segment map covers 0–9, minus, space, E, r, o (for "Error" display)

Step 2 Button Keyboard & Calculator Logic
Add a keyboard providing all numbers and typical buttons (+ - / * = etc) and a few advanced functions.

What was built

Button Layout

A 5-column CSS grid with 6 rows of buttons, each visually coded by function:

RowButtons
1MC, MR, M+, M−, AC
2√, x², 1/x, %, CE
37, 8, 9, ÷, ←
44, 5, 6, ×, ±
51, 2, 3, −, = (spans 2 rows)
60 (wide), ., +
5 Visual Button Styles
  • btn-num — dark grey for number keys
  • btn-op — purple for operators (+, −, ×, ÷)
  • btn-eq — accent blue for equals
  • btn-fn — dark blue-grey for advanced functions
  • btn-clr — red-tinted for AC / CE

All buttons have a 3D press effect using translateY + box-shadow shift on :active.

Calculator Engine
  • Standard input chaining: number → operator → number → equals
  • Operator chaining: 5 + 3 × evaluates 5 + 3 immediately, stages 8 ×
  • formatNumber() caps output to 20 digits, falls back to exponential notation
  • Error handling for divide-by-zero, negative sqrt, and overflow
Advanced Functions
  • √ — square root (errors on negative input)
  • x² — square the current value
  • 1/x — reciprocal (errors on zero)
  • % — context-aware: 200 + 10% = 220, standalone 50% = 0.5
  • ± — toggle sign
  • ← — backspace (delete last digit)
  • MC / MR / M+ / M− — memory register
  • CE — clear entry only; AC — full reset
Keyboard Support

Physical keyboard input is mapped: 0–9, . + - * / Enter/=, Backspace, Escape, %

Key code: Calculator state machine 
let currentInput = '0';    // what the user is typing
let previousValue = null;  // left operand
let operator = null;       // pending operator
let resetNext = false;     // next digit replaces display
let memory = 0;            // memory register

function calculate(a, op, b) {
  switch(op) {
    case '+': return a + b;
    case '-': return a - b;
    case '*': return a * b;
    case '/': return b === 0 ? Infinity : a / b;
  }
}
