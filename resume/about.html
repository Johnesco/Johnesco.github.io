About This Resume System

This is a dynamic, JavaScript-powered resume website. All resume content is stored in a single JSON data file and rendered dynamically, allowing for multiple output formats, URL-based filtering, and ATS-optimized PDF output.

Key Features

Single Source of Truth

All resume data lives in one JSON file. Update once, changes appear everywhere.

Multiple Formats

Styled HTML, plain text (ATS-friendly), visual timeline, and Bootstrap versions.

URL Filtering

Filter by years, industry tags, or skills using URL query parameters.

ATS-Optimized Output

Print/PDF uses standard headers, single-column layout, and left-aligned content.

Resume Layout

Section Headers

The resume uses ATS-standard section headers for reliable parsing:

  • Skills - Skill categories with keyword tags
  • Professional Experience - Recent work history with full details
  • Additional Experience - Condensed older jobs (expandable on click)
  • Education - Schools and training

Job Entry Format

Each job displays in ATS-standard order:

  1. Job Title - Most prominent (as heading)
  2. Company Name | Location
  3. Date Range
  4. Summary and bullet points

Additional Experience (Expandable)

When a profile with earlier experience years is active, older jobs appear in a condensed "Additional Experience" section:

  • Jobs display as one-liners with a + icon
  • Click any job to expand and see its full summary and highlights
  • Click again to collapse
  • Print/PDF keeps all jobs collapsed for compact ATS-friendly output

Print/PDF Output

When you print or save as PDF, the resume automatically optimizes for ATS parsing:

  • Single-column layout - All content flows in correct reading order
  • Left-aligned content - Contact info and dates align left, not right
  • Skills as text - Skill pills convert to comma-separated inline text
  • Black text - All colors forced to black for reliable extraction
  • Compact spacing - Tighter margins for efficient page use
  • Hidden expand icons - Additional Experience stays condensed

URL Query Parameters

Filter the resume dynamically by adding query parameters to the URL:

Parameter Description Example
profile Select a predefined profile (qa-lead, business-analyst, instructor, all) ?profile=qa-lead
years Override work history years (full-detail window) ?years=10
additional Override additional condensed years beyond work history ?additional=5
format Skills display format (tags or list) ?format=list

Example URLs

  • Last 10 years only: index.html?years=10
  • Show all jobs: index.html?profile=all
  • Business analyst profile: index.html?profile=business-analyst
  • Combined filters: index.html?profile=qa-lead&years=15
  • Add 5 years of condensed history: index.html?additional=5

Available Tags

Jobs and skills are tagged for profile-based filtering. Each profile has a tags array in resume-config.js listing which tags it includes. Jobs/skills appear in a profile if any of their tags overlap with the profile's tags array:

Profile Tags

qa-lead, business-analyst, instructor

Other Tags

event-host, design, security, administrative

Technical Details

  • No Build Process - Static files, edit and refresh
  • Vanilla JavaScript - No framework dependencies (except Bootstrap version uses jQuery)
  • CSS Custom Properties - Colors and spacing defined as variables
  • ATS-Optimized Print - Media queries transform layout for PDF output
  • Expandable UI - Click-to-expand older jobs without page reload
  • Profile-Driven Defaults - Default profile (qa-lead) controls work history years, skills format, and condensed history

Data Format

The resume data follows a modified JSON Resume schema with these sections:

  • basics - Name, title, contact info, summary
  • work - Array of job objects with tags for filtering
  • education - Array of educational institutions
  • skills - Array of skill categories with keywords
  • languages - Array of spoken languages

Updating Content

To update resume content:

  1. Edit js/resumeJSON.js
  2. All pages automatically reflect changes on next load
  3. No compilation or build step required

To change default settings (years, profiles, etc.):

  1. Edit js/resume-config.js
  2. Modify defaultProfile, defaultWorkHistoryYears, or profile definitions
