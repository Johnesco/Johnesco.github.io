Comprehensive Guide

The ifMUD help system is organized into topics, commands, flags, fields, and a building tutorial. Type help <topic> in-game for any of these.

Basic Commands (say, emote, look, read, examine, etc.)
# - Comment prefix
Lines beginning with # are treated as comments and ignored by the MUD.
; - Channel shortcut
The semicolon command is used to talk on channels with precision, avoiding ambiguity when multiple channels share a prefix. ;channelname message
say
Speak to everyone in your current room. Can be abbreviated to a double-quote. say Hello everyone! "Hello everyone! Others see: YourName says "Hello everyone!"
emote
Express actions or emotions. Can be abbreviated to a colon. emote waves happily. :waves happily. Others see: YourName waves happily.
look
Look at your current location, showing the room description, exits, and contents. look look at painting look north
read
Read text on an object (if it has readable text set). read sign
examine
Get detailed information about an object, including its properties. examine painting
inventory
Show what you're carrying. Shortcut: i
get / take
Pick up an object. get lamp take sword
drop
Put down an object you're carrying. drop lamp
give
Give an object to another player. give sword to Bob
home
Instantly teleport to your designated home location.
quit
Disconnect from ifMUD.
again
Repeat your last command. Shortcut: g
Communication (whisper, page, channels)
whisper
Speak privately to someone in the same room. Shortcut: . (period) whisper Bob = I have a secret .Bob I have a secret whisper Bob, Alice = Hi both To privately pose, start with a colon: .Bob :winks
page
Send a private message to anyone on the MUD, regardless of location. page Bob = Are you coming to the meeting?
channels
Channels allow group communication across the entire MUD. Use the channel name followed by your message: gweep Hello everyone! For emotes on channels: gweep :waves See also: @joinchannels, @leavechannels, @statchannel, @listchannels
channel-aliases
Channel aliases allow you to create shortcuts for channel names. See help channel-aliases in-game.
channel-flags
Channel flags control channel behavior: private, moderated, topiclock, etc. See help channel-flags in-game.
@joinchannels
Join one or more channels. @joinchannels mudding @joinchannels foo bar baz
@leavechannels
Leave one or more channels. @leavechannels mudding
@listchannels
List all available channels on the MUD.
@statchannel
Show details about channels. Without arguments, lists channels you're on with their topics. @statchannel - Your channels @statchannel gweep - Details about gweep @statchannel b* - All channels starting with 'b' Popes (admins) show with +, cardinals with -.
@mutechannels / @unmutechannel
Temporarily mute a channel without leaving it. @mutechannels gweep @unmutechannel gweep
@makechannel
Create a new channel. Creator becomes the "pope" (admin). @makechannel mychannel @makechannel mychannel = private @makechannel mychannel = moderated private @makechannel #games/mychannel - In a category
@paste / @endpaste
Enter paste mode to send multiple lines with a prefix. @paste #peanut-gallery Line one Line two @endpaste
smartresolve
Option that restricts channel searches to recently-used channels, reducing ambiguity errors.
Bulletin Board (bb commands)
bb
The bulletin board system for community announcements and discussions.
bb scan
Check for new/unread messages on the bulletin board.
bb list
List all messages on the bulletin board. bb list bb list 20 - Last 20 messages
bb read
Read a specific message. bb read 42
bb range
Read a range of messages. bb range 40-50
bb post
Post a new message to the bulletin board. Follow the prompts for subject and body.
bb delete
Delete one of your messages. bb delete 42
bb undelete
Restore a deleted message. bb undelete 42
bb modify
Edit one of your messages. bb modify 42
bb clear
Mark all messages as read.
bb check
Check for new messages.
bbnotify
Control bulletin board notifications: @field me = bbnotify : always - Login and while connected @field me = bbnotify : logon - Only on login @field me = bbnotify : update - Only while connected @field me = bbnotify : never - No notifications
Player Information (who, where, finger, @doing)
who
List all connected players with their @doing messages.
where
Find where a specific player is located. where Bob
@users
Detailed list of connected users.
finger
View player profile information: last connection, real name, email, URL, IF work, plan. finger Bob
@doing
Set what appears next to your name in the who list. @doing Writing some JotaCode
@poll
Set the poll question at the top of the who list. @poll Your favorite Infocom game?
@away
Set an away message shown to anyone who pages or whispers to you. @away Getting coffee, back in 5
@field
Set profile fields: @field me = rname : Your Real Name @field me = address : your@email.com @field me = url : https://yoursite.com @field me = game : Your IF Game @field me = plan : Currently working on...
Building Commands (@dig, @create, @describe, @link, etc.)
@dig
Create a new room. @dig My New Room
@create
Create a new object. @create sword
@describe
Set the description of an object or room. @describe here = A cozy room with a fireplace. @describe sword = A gleaming blade.
@name
Rename an object. Use semicolons for aliases. @name sword = Excalibur;sword;blade @name here = The Great Hall
@link
Link an object or exit to a destination, or set your home. @link me = here - Set home @link north = #123 - Link exit to room
@open
Create an exit from the current room. @open north;n
@action
Create an action (verb) attached to an object. @action push;press = button
@set
Set flags on objects. @set sword = dark @set here = abode
@lock
Set a lock on an object or exit. @lock door = gold key - Requires carrying the key @lock door = *Bob - Only Bob can use it @lock door = *Bob | gold key - Bob OR key holder @lock door = !*Bob - Anyone except Bob
@success / @osuccess
Messages shown when someone passes a lock. @success door = You open the door. @osuccess door = opens the door.
@fail / @ofail
Messages shown when someone fails a lock. @fail door = The door is locked. @ofail door = tries the door but it's locked.
@drop / @odrop
Messages shown when an object is dropped (or when exiting through an exit). @drop sword = You set down the sword carefully. @odrop sword = sets down a sword.
@teleport
Teleport yourself or an object to a location. @teleport me = #123 @teleport sword = here
@find
Search for objects by name. @find sword
@recycle
Delete an object you own. @recycle sword
@chown
Change ownership of an object (requires permission).
@clone
Create a copy of an object. @clone sword
Flags (abode, dark, quiet, etc.)
abode
If a location is set abode, anyone can set their home to that location. @set here = abode
dark
Objects set dark don't appear in room descriptions. Dark rooms don't show contents.
quiet
Quiet locations don't announce arrivals/departures. Quiet players don't announce connects/disconnects. Quiet objects don't show success/fail messages.
examinable
Allows others to examine the object's properties.
chownok
Allows others to @chown the object to themselves.
jumpok
Allows others to @teleport to this location.
linkok
Allows others to @link exits to this location.
visible
Object is visible even in dark rooms.
slippery
Object cannot be picked up.
zoned
Player is "zoned out" and will show away messages.
Building Tutorial (t-concepts, t-rooms, t-exits, t-objects, t-locks, etc.)
tutorial
Start here: help tutorial in-game for the building tutorial overview.
t-concepts
Basic MUD concepts: rooms, objects, exits, players, and database references (dbrefs).
t-rooms / t-rooms2 / t-rooms3
Creating and describing rooms with @dig and @describe.
t-exits / t-exits2 / t-exits3
Creating exits with @open, linking with @link, and creating two-way connections.
t-objects / t-objects2 / t-objects3
Creating objects with @create, describing them, and setting properties.
t-locks / t-locks2 / t-locks3 / t-locks4
Locking objects and exits:
  • @lock door = key - Requires carrying key
  • @lock door = *Bob - Only player Bob
  • @lock door = key | *Bob - Key OR Bob (| = OR)
  • @lock door = key & badge - Key AND badge (& = AND)
  • @lock door = !*Bob - NOT Bob (anyone except Bob)
  • Use parentheses for complex locks: (key | *Bob) & !*Evil
t-verbs / t-verbs2 / t-verbs3
Creating custom verbs with @action and setting success/fail messages.
t-nouns / t-nouns2

JotaCode Programming (functions, substitutions, variables)
programming
Overview of JotaCode, ifMUD's programming language for dynamic content.
functions
List of available JotaCode functions. See help functions in-game.
substitutions / subs-list
Text substitutions like %n (player name), %s (subjective pronoun), %o (objective pronoun), etc.
jotavars
JotaCode variables:
  • *self / *this - The object running the code
  • *player / *me - The player executing the command
  • *field / *value - For @fieldloop
  • *nowhere / *home - Special location constants

Apartments & Personal Space
apartments
Free apartment space is available at Edifice Towers. To get there: xyzzy, then south three times. Each floor has 10 exits that can be claimed.
apartments2
Instructions for claiming apartment space. See help apartments2 in-game.
Setting your home
Once you have a room (or are in an ABODE room): @link me = here Now home will teleport you there.
homemsg / ohomemsg
Customize messages when you go home: @field me = homemsg : vanishes in a puff of smoke @field me = ohomemsg : appears in a puff of smoke
