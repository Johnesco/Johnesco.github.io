Process 1:

Store each page type in the following format

var ideaPages = {
	"sectionName": "Ideas Pages",
	"pre": "www",
	"sub": "ideas/",
	"endPoints": [
		"hot-products",
		"halloween"
	]
 };

 var searchPages = {
	"sectionName": "Search Pages",
	"pre": "www",
	"sub": "s/",
	"endPoints":[
		"bacon"
    ]
 };

These are kept separate and not in one object to facilitate editing

Process 2:

Stitch these together into one large object at load time

sectionName becomes the property name

{
  "Search Pages": {
    "pre": "www",
    "sub": "s/",
    "endPoints": [
      "bacon"
    ]
  },
  "Ideas Pages": {
    "pre": "www",
    "sub": "ideas/",
    "endPoints": [
      "hot-products",
      "halloween"
    ]
  }
}

Process 3:

Properties from the HTML page form are combined with those from the object

From Form

  • security (http:// vs https://)
  • environment (rmntest.com, rmnstage.com, retail)
  • slice (control, other, etc..)

From Object

  • prefix (www, m, giftcard)
  • optional subdirectory (ideas, coupon, s)
  • array of endpoints

The will be combined as..

security + environment + subdirectory + endpoint for each endpoint in array

Process 4:

Result produced in Process 3 is rendered to page

Process 5:

Any change to properties on form starts again at process 2
